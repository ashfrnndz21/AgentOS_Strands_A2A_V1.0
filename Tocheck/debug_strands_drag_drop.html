🐛 Strands Drag & Drop Debug Guide

Debug guide to identify and fix utility node drag & drop issues

🔍 Debug Steps

  1. Open Browser Console: Press F12 or right-click → Inspect → Console
  2. Navigate to Strands Workflow: Go to Multi-Agent Workspace → Strands Intelligent Workflow
  3. Try Dragging Utilities: Go to Utilities tab and try dragging each node
  4. Check Console Messages: Look for the debug messages we added

🎯 Expected Console Messages

When Starting Drag (from AgentPalette):

🚀 Starting drag for utility node: { type: "utility-node", nodeType: "decision", // or handoff, human, etc. nodeData: { name: "Decision", description: "...", ... } }

When Dropping on Canvas (from StrandsWorkflowCanvas):

🎯 Drop data received: { type: "utility-node", nodeType: "decision", nodeData: { ... } } ✅ Creating utility node: decision

❌ Common Issues & Solutions

Issue 1: No drag start message

Problem: Utility nodes are not draggable

Solution: Check if draggable="true" is set on utility nodes

Issue 2: Drag starts but no drop message

Problem: Drop zone not accepting drops

Solution: Check onDragOver and onDrop handlers

Issue 3: Drop message but no node creation

Problem: Node creation methods failing

Solution: Check orchestrator methods and node type mapping

Issue 4: Node created but not visible

Problem: Node component not rendering

Solution: Check node types mapping and component imports
